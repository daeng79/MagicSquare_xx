# 4x4 마방진 Mom Test 인터뷰 보고서

## 1. 문서 목적

- MagicSquare_xx 프로젝트의 **문제 인식**을 Rob Fitzpatrick *The Mom Test* 방식으로 검증한 인터뷰 결과를 정리한다.
- 미래 의향·솔루션 찬성이 아닌 **과거 사실·행동·손실**만을 근거로 진짜 문제를 도출한다.

## 2. 인터뷰 개요

| 항목 | 내용 |
|---|---|
| 진행 방식 | Mom Test 스타일 1:1 인터뷰 (질문 1개씩) |
| 페르소나 | 4×4 부분 마방진을 손으로/코드로 다루는 학습자 |
| 금지 질문 | "만들면 좋겠어?", "이 기능 필요해?", "TDD 하면 어때?" |
| 허용 질문 | "마지막으로 ~했을 때", "그때 몇 분 걸렸어?", "뭐 때문에 포기했어?" |

## 3. 페르소나

- 4×4 부분 마방진(빈칸 2개) 실습을 수행하는 **학습자**
- ECB 분류·설계 연습과 함께, 빈칸 숫자를 맞추는 **문제 해결 단계**를 경험함
- 알고리즘 난이도보다 **제약 조건을 동시에 만족시키는 판단**에서 어려움을 겪음

## 4. 인터뷰 전문 (Q&A)

### Q1. 마지막으로 4×4 마방진에서 빈칸 2개를 맞춰 본 경험

**A.** 첨부 자료(4×4 부분 마방진 + ECB 개념 분석)와 같이 작업했으나 어려움이 있었다.

### Q2. (추궁) 막힌 순간이 ECB 분류 / 빈칸 맞추기 / 실패 조건 도출 중 어디였는가?

**A.** **빈칸 2개 숫자를 맞추는 단계**였다.

### Q3. 처음 넣어본 숫자 조합과 확신 여부

**A.** 기억나지 않는다.

### Q4. (추궁) 행·열·대각선 중 어디부터 확인했는가?

**A.** **대각선부터** 막혔다.

### Q5. 대각선에서 무엇이 안 맞았는가?

**A.** **34로 맞추려다 합이 안 맞아** 진행이 막혔다.

### Q6. 합이 안 맞는다는 것을 알았을 때 다음 행동

**A.** **그 순간 실습을 종료**했다.

### Q7. (추궁) 종료는 잠시 쉰 것인가, 다시 안 열 생각인가?

**A.** 실습을 **거기서 끊었고**, **다시 열 생각이 없었다** ("이 문제는 나랑 안 맞다").

### Q8. (추가 확인) 비슷한 과제 재시도 및 손실

**A.** 비슷한 과제를 **다시 연 적 없다**. **20분**의 시간을 날렸다.

## 5. 확인된 과거 사실 (Mom Test 증거)

| # | 과거 사실 | 신호 강도 |
|---|---|---|
| 1 | ECB 분류가 아닌 **빈칸 맞추기**에서 막힘 | 행동 |
| 2 | **대각선 합 34** 맞추기에서 진행 중단 | 구체적 맥락 |
| 3 | 합 불일치 후 **즉시 실습 종료** | 행동 |
| 4 | **재개 의사 없음** | 강한 포기 신호 |
| 5 | 비슷한 과제 **재시도 없음** | 패턴 |
| 6 | **20분** 시간 손실 | 비용(시간) |

## 6. Mom Test 체크리스트

| 항목 | 상태 | 비고 |
|---|---|---|
| 과거 사실 | ✅ | 막힘 → 종료 → 재개 안 함 |
| 구체적 손실 | ✅ | 20분 |
| 돈을 쓴 행동 | ❌ | 미확인 |
| 미래 의향("~하면 좋겠다") | ❌ | 사용하지 않음 |
| 솔루션 찬성 | ❌ | 사용하지 않음 |
| 칭찬/일반론 | ❌ | 사용하지 않음 |

## 7. 문제 정의

### 7.1 표면 문제 (잘못된 정의 — 솔루션 섞임, 금지)

- "4×4 마방진 **정답을 자동으로 만들어주는 솔버**가 필요하다."
- "**대각선 검증 기능**이 필요하다."
- "**TDD로 Solver**를 만들면 된다."

### 7.2 진짜 문제 (한 문장)

**여러 제약(행·열·대각선)이 겹친 상태에서, 중간에 어디가 틀렸는지·다음에 무엇을 확인할지 판단할 수 없어 20분을 쓰고도 진행을 이어가지 못하고, 같은 유형의 과제를 다시 열지 않게 된다.**

### 7.3 문제 분해

| 관찰 | 해석 |
|---|---|
| 대각선부터 막힘 | 확인 **순서·우선순위**가 고정되지 않음 |
| 합이 34가 아님 | "거의 맞음"과 "틀림"을 **중간에 구분 못함** |
| 즉시 종료 | **다음 확인 단계**를 모름 → 진행 불가 |
| 재개 안 함 | 일회성 좌절이 **학습 중단 패턴**으로 고정됨 |

## 8. 기존 문제정의 보고서와의 정합성

Report/01(문제 인식)에서 도출한 내용과 본 인터뷰 결과는 동일 축이다.

| Report/01 개념 | Mom Test 인터뷰 증거 |
|---|---|
| "거의 맞음" 착시 | 대각선 합 불일치 후에도 어디가 틀렸는지 특정 못 함 |
| 판단 기준의 재현성 | 20분 후 포기·재시도 없음 — 기준이 없으면 학습 지속 불가 |
| 실패를 구조적 신호로 | "합이 안 맞음"만 알고 **어떤 제약 위반인지** 추적 못 함 |
| 조건 중심 문제 정의 | "정답 찾기"가 아니라 **막혔을 때 판단 체계 부재**가 핵심 |

## 9. MagicSquare_xx 프로젝트 함의 (문제만, 솔루션 제외)

- 프로젝트가 다루는 핵심은 **"빈칸 두 칸의 정답"** 자체가 아니다.
- **막혔을 때 어디가 깨졌는지를 구조적으로 드러내지 못하면**, 학습자는 20분 뒤 실습을 끊고 돌아오지 않는다.
- 따라서 요구사항·테스트·실패 조건은 **"정답 반환"**과 동등하게 **"위반 조건의 명시적 드러냄"**을 다루어야 한다.

## 10. 미확인 항목 (후속 인터뷰 후보)

- 첫 시도 숫자 조합 (응답자 기억 없음)
- 행·열은 맞았는지, 어느 대각선인지
- 20분 중 어느 구간에 대부분 소비했는지
- 실습 종료 후 ECB 분류·실패 조건 도출은 진행했는지

## 11. Mom Test 워크북 (최종본)

### 페르소나

4×4 부분 마방진을 손으로/코드로 다루는 학습자. ECB·설계 연습과 함께 빈칸 맞추기를 수행하며, **다중 제약 판단**에서 어려움을 겪는다.

### 인터뷰 답변 (과거 사실)

1. ECB 분류보다 **빈칸 2개 숫자 맞추기**에서 막혔다.
2. **대각선 합을 34로** 맞추려다 합이 안 맞아 진행이 멈췄다.
3. 그 순간 **실습을 종료**했고, **다시 열 생각이 없었다**.
4. 비슷한 과제를 **다시 연 적 없다**.
5. **20분**의 시간을 날렸다.

### 표면 문제 (금지 — 솔루션 섞인 문장)

- "4×4 마방진 정답을 자동으로 만들어주는 TDD 솔버가 필요하다."
- "ECB 3레이어 + pytest로 마방진을 풀어주는 프로그램이 필요하다."

### 진짜 문제 (한 문장)

**여러 제약(행·열·대각선)이 겹친 상태에서, 중간에 어디가 틀렸는지·다음에 무엇을 확인할지 판단할 수 없어 20분을 쓰고도 진행을 이어가지 못하고, 같은 유형의 과제를 다시 열지 않게 된다.**
